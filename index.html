Technical Notes

可阅读的技术博客

每篇笔记聚焦一个工程问题：输入、约束、实现路径和验证点。

字符 LCD 的核心限制不是分辨率，而是可自定义字符数量和刷新时序。LCD1602/2004 只能同时保留少量 CGRAM 字形，因此动画需要在“图形资源复用”和“刷新频率”之间折中。

  • 输入：PC13 按键通过 EXTI 触发，避免在主循环中轮询阻塞。
  • 时序：定时器中断推进游戏 tick，主循环只处理状态更新和显示提交。
  • 渲染：将动态图形拆成少量可复用字符块，用状态机控制角色、障碍物和滚动窗口。
  • 验证：观察按键响应、动画抖动、CGRAM 更新频率和 LCD 命令延迟。

包头插入模块的风险集中在边界时序：header 与 payload 的首拍对齐、keep 字节有效掩码、last 信号传播，以及 ready 反压下的缓存一致性。

  • 接口：valid/ready 决定传输是否发生；keep 描述每拍有效字节；last 标记包尾。
  • 状态：inserting_header、byte_cnt、buffer_valid 用于跟踪插入阶段和暂存数据。
  • 测试：覆盖 header 长度为 0、非整拍、整拍、多拍，以及 ready_out 拉低的场景。
  • 输出检查：data_out 顺序、keep_out 掩码、last_out 时刻和 ready_in 反压行为。

仿真自动化交付需要检查求解状态和结果对象，而不是只检查 Workbench 项目是否保存。一个可复核的 Mechanical 结果至少应包含网格、载荷、边界条件、求解完成状态和后处理数值。

  • 前处理：几何单位、材料、约束、载荷方向和接触设置。
  • 求解：solver 状态、错误/警告日志、收敛或静力平衡信息。
  • 后处理：contour、legend、Min/Max 数值、结果对象 evaluated 状态。
  • 交付：项目文件、截图、结果摘要和可重复执行的脚本入口。

Sky Monitor API 将天文计算拆成三层：本地星表查询、行星星历计算、面向前端的极坐标投影。这样前端不需要理解天体力学，只消费 chart_x、chart_y、radius 和解释文本。

  • 星表：HYG Database 构建 SQLite 索引，按亮度和高度角筛选。
  • 行星：Skyfield 读取 JPL DE421 星历，计算观测时间下的高度角与方位角。
  • 深空天体：Messier/NGC 元数据补充类型、名称和观测建议。
  • 输出：JSON 同时服务星图绘制和 LLM 观测建议生成。
